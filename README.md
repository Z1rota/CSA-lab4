## Язык программирования Asm (Stack)

Синтаксис языка представляет собой ассемблер, адаптированный под стековую архитектуру. Код разделяется на секции данных и текста.

Форма Бэкуса-Наура (БНФ):
```
<program> ::= <section_data> <section_text>
<section_data> ::= ".data\n" <data_decl>*
<data_decl> ::= <label> ":" <directive> <value> "\n" | <org_directive>
<directive> ::= ".word" | ".pstr"
<section_text> ::= ".text\n" <code_line>*
<code_line> ::= [<label> ":"] <instruction> [<operand>] | <macro_def> | <org_directive>
<macro_def> ::= "%define" <name> <value>
<org_directive> ::= ".org" <address>
```


**Особенности семантики:**
- Поддерживаются макросы `%define` для констант (разрешаются на этапе трансляции).
- Строки хранятся в формате Pascal strings (`.pstr`): первое слово — длина строки, последующие слова — символы (по одному символу в 32-битном слове).
- Архитектура стековая: математические операции операндов не принимают, а берут значения с вершины стека (TOS) и кладут результат обратно.
- Директива `.org` позволяет размещать код и данные по абсолютным адресам.

## Организация памяти

- **Архитектура:** Фон-Неймана.
- **Машинное слово:** 32 бита, знаковое целое.
- **Адресное пространство:** Линейное. Память представляет собой массив из 2048 слов.
- **Аппаратные стеки:** Процессор содержит два независимых аппаратных стека:
  - `Data Stack` (Стек данных) — используется для вычислений (глубина 256).
  - `Return Stack` (Стек возвратов) — используется для сохранения PC при вызове процедур (`CALL`) и обработке прерываний (`Trap`).
- **Прерывания:** Вектор прерываний жёстко задан по адресу `0x0010`.

## Система команд

Инструкции упаковываются в одно 32-битное машинное слово:
- Старшие 8 бит: `Opcode` (код операции).
- Младшие 24 бита: Операнд (знаковое число).

### Набор инструкций

Для описания семантики команд используются следующие обозначения:
- `DS` (Data Stack) — аппаратный стек данных (используется для АЛУ).
- `RS` (Return Stack) — аппаратный стек возвратов (для `CALL`, `RET` и прерываний).
- `MEM` — общая память (массив машинных слов).
- `PC` — счётчик команд (Program Counter).
- `PORT` — адресное пространство портов ввода-вывода.
- `IE` — флаг разрешения прерываний (Interrupt Enable).

| Мнемоника | Опкод | Действие (Семантика) | Операнд (`arg`) |
|-----------|-------|----------------------|-----------------|
| `NOP`     | 0x00  | Нет операции | Игнорируется |
| `PUSH`    | 0x01  | `DS.push(arg)` | Число (константа) |
| `PUSH_M`  | 0x02  | `DS.push(MEM[arg])` | Адрес памяти |
| `POP_M`   | 0x03  | `MEM[arg] <- DS.pop()` | Адрес памяти |
| `ADD`     | 0x04  | `b = DS.pop(); a = DS.pop(); DS.push(a + b)` | Игнорируется |
| `SUB`     | 0x05  | `b = DS.pop(); a = DS.pop(); DS.push(a - b)` | Игнорируется |
| `MOD`     | 0x06  | `b = DS.pop(); a = DS.pop(); DS.push(a % b)` | Игнорируется |
| `CMP`     | 0x07  | `b = DS.pop(); a = DS.pop(); DS.push(a == b ? 1 : 0)` | Игнорируется |
| `GT`      | 0x13  | `b = DS.pop(); a = DS.pop(); DS.push(a > b ? 1 : 0)` | Игнорируется |
| `MUL`     | 0x11  | `b = DS.pop(); a = DS.pop(); DS.push(a * b)` | Игнорируется |
| `DIV`     | 0x12  | `b = DS.pop(); a = DS.pop(); DS.push(a / b)` | Игнорируется |
| `JMP`     | 0x08  | `PC = arg` | Адрес перехода |
| `JZ`      | 0x09  | `if (DS.pop() == 0) { PC = arg }` | Адрес перехода |
| `CALL`    | 0x0A  | `RS.push(PC); PC = arg` | Адрес процедуры |
| `RET`     | 0x0B  | `PC = RS.pop()` | Игнорируется |
| `IN`      | 0x0C  | `DS.push(PORT[arg])` | Номер порта |
| `OUT`     | 0x0D  | `PORT[arg] <- DS.pop()` | Номер порта |
| `IRET`    | 0x0E  | `PC = RS.pop(); IE = 1` | Игнорируется |
| `EI`      | 0x10  | `IE = 1` | Игнорируется |
| `HALT`    | 0x0F  | Остановка тактового генератора | Игнорируется |

### Ввод-вывод 
Используется система прерываний с аппаратным таймером (расписанием):
- При наступлении заданного такта, внешний контроллер кладёт символ в буфер и дёргает линию прерывания.
- Процессор аппаратно сохраняет `PC` на `Return Stack`, сбрасывает флаг `IE` (Interrupt Enable) и переходит по вектору `0x0010`.
- Ввод/вывод маппится на порты:
  - Порт `0`: Чтение символа.
  - Порт `1`: Символьный вывод.
  - Порт `2`: Числовой вывод.

### Суперскалярность
В стековой архитектуре все операции проходят через вершину стека (TOS), что делает классическое суперскалярное исполнение (распараллеливание двух математических команд) невозможным из-за RAW-зависимостей (Read-After-Write). Чтобы реализовать параллельное выполнение без превращения процессора в CISC, применяется паттерн **Instruction Folding (Свертывание инструкций)** с использованием аппаратного **теневого регистра (`TOS_SHADOW`)**.

Теневой регистр позволяет аппаратно разделить вычислительный процесс в АЛУ и работу с медленной памятью:
1. **Аппаратный SWAP (Отложенное сохранение):** При сохранении переменной в память (`POP_M`), результат не выгружается в память сразу. за 1 такт выполняется аппаратный SWAP — значение прячется в `TOS_SHADOW`, освобождая стек для новых вычислений.
2. **Фоновая запись:** Тракт данных (DataPath) выполняет выгрузку в ОЗУ из теневого регистра  **параллельно** с вычислениями в АЛУ над следующими данными.
3. **Обратный SWAP (Bypass / Forwarding):** Если сохраненная переменная снова нужна для вычислений (`PUSH_M`), а она всё ещё лежит в теневом регистре, выполняется обратный SWAP. Это полностью исключает этап медленного чтения из ОЗУ.

**Control Unit:**
Control Unit производит динамическое планирование (выборку 2-х инструкций) и разрешает зависимости:
- **Разделение ресурсов:** Если первая инструкция задействует АЛУ, а вторая осуществляет фоновую запись (`POP_M`) или переход (`JMP`/`JZ`), они сливаются (Folding) и выполняются параллельно за один такт.
- **Блокировка при конфликтах:** Если выявляется конфликт по шине (например, АЛУ запрашивает чтение новой переменной `PUSH_M`, в то время как теневой регистр ещё пишет предыдущую), Control Unit аппаратно блокирует параллельное исполнение и переводит конвейер в режим простоя до освобождения памяти.

## Транслятор

Интерфейс командной строки: `python translator.py <source.asm> <output.bin>`

Транслятор реализован в два прохода:
1. **Pass 1:** Удаление комментариев, резолвинг макросов `%define`, вычисление адресов меток, генерация данных (`.word`, `.pstr`) в память с учётом директив `.org`.
2. **Pass 2:** Подстановка адресов вместо меток, упаковка инструкций (Opcode + Arg) в 32-битные бинарные слова. Вывод в бинарный файл (`.bin`) и человекочитаемый лог (`.bin.log`).

## Модель процессора

Интерфейс командной строки: `python machine.py <code.bin> <schedule.txt>`

Архитектура разделена на **Control Unit (Диспетчер)** и **Data Path (Тракт данных)**. Процессор поддерживает потактовое исполнение (tick-by-tick) и аппаратную суперскалярность.

### Схема процессора (с теневым регистром)

Для реализации суперскалярности в стековой архитектуре без блокировок АЛУ используется аппаратный **теневой регистр (TOS_SHADOW)** для фоновой записи в память.

```text
       Interrupt Timer (Schedule)
             |
             v
      +---------------+         [Port 0 IN] --->+
      | Interrupt     |---> IE                  |
      | Controller    |---> Trap Vector (0x10)  |
      +---------------+                         v
                                         +-------------+
            +--------------------------->| ALU / Stack |
            |            +----+          | Execution   |---> [Port 1 Char OUT]
            |      +---->| SP |<-------->| Unit 1      |
            |      |     +----+          +-------------+---> [Port 2 Int OUT]
            |      |                            |
            |      |                       (1 такт SWAP / Bypass)
            |      |                            v
            v      v                     +-------------+
    +--------------------+     (Фоновая  | TOS_SHADOW  |
    |                    | <=============| Register    |
+-->|   Shared Memory    |     запись    +-------------+
|   |   (Code + Data)    |     2 такта)         ^
|   +--------------------+                      |
|            |      | (Fetch 64-bit: 2 Instructions)
|            v      v
|     +--------------------+
|     | Instruction Queue  |
|     +--------------------+
|          |         |
|      Inst 1       Inst 2
|          |         |
|          v         v
|     +--------------------+
|     | Control Unit       |---> (Анализ конфликтов и зависимостей)
|     |                    |
|     +--------------------+
|               | (Jumps / Control)
|               v
|     +--------------------+
|     | Control Execution  |
|     | Unit 2             |
|     +--------------------+
|        |           ^
|        v           |
|      +----+      +----+
+------| PC |      | RP | (Return Pointer / Stack)
       +----+      +----+
````
